# Implementation Plan: Course Companion FTE — Phase 2 (Hybrid Intelligence)

**Branch**: `001-hybrid-intelligence` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)

## Summary

Add two Pro-tier premium endpoints — LLM-Graded Assessment and Cross-Chapter Synthesis — on top of the existing Phase 1 FastAPI backend. All new code lives exclusively in `backend/app/premium/`. Phase 1 files are not modified (except 4-line router registration in `main.py`). LLM calls use Anthropic SDK (`claude-sonnet-4-20250514`) with structured output via `tool_use`. Every call is logged to a new `llm_usage` table. Rate limited to 10 calls/user/day via DB query.

## Architecture Decisions

### AD-1: Premium Router Registration in main.py

```python
# app/main.py — conditional registration (HYBRID routes)
if settings.ANTHROPIC_API_KEY:
    from app.premium.routers import assessment, synthesis, usage
    app.include_router(assessment.router, prefix="/premium", tags=["premium"])
    app.include_router(synthesis.router, prefix="/premium", tags=["premium"])
    app.include_router(usage.router, prefix="/premium", tags=["premium"])
```

Phase 1 tests run without `ANTHROPIC_API_KEY` — premium modules never imported, zero-LLM audit still passes.

### AD-2: Pro Tier Gate — Shared Dependency

```python
# app/premium/__init__.py
async def require_pro(current_user = Depends(get_current_user)) -> User:
    if current_user.tier != "pro":
        raise HTTPException(
            status_code=403,
            detail="This feature requires Pro plan. Upgrade at /pricing"
        )
    return current_user
```

### AD-3: LLM Call Pattern with tool_use

```python
# HYBRID — LLM CALL
response = client.messages.create(
    model=settings.LLM_MODEL,
    max_tokens=settings.MAX_TOKENS_ASSESSMENT,
    system=prompt.format(chapter_id=chapter_id, chapter_content=chapter_content),
    tools=[ASSESSMENT_TOOL_SCHEMA],
    tool_choice={"type": "tool", "name": "submit_assessment"},
    messages=[{"role": "user", "content": f"Question: {question}\n\nAnswer: {student_answer}"}],
)
result = response.content[0].input   # guaranteed valid JSON
await log_usage(db, user_id, "assessment", response.usage)
```

`tool_use` forces Anthropic to return JSON matching the declared schema — eliminates 502 parsing errors.

### AD-4: Structured Output Schemas

**Assessment tool** (`submit_assessment`):
- `score`: integer 0–100
- `feedback`: string
- `strengths`: array of strings
- `improvements`: array of strings
- `suggested_reading`: string

**Synthesis tool** (`submit_synthesis`):
- `synthesis`: string
- `key_connections`: array of strings (each must cite `[chapter-XX]`)
- `knowledge_graph`: array of `{from, to, relationship}` objects
- `recommended_next`: string (chapter_id)

### AD-5: Rate Limiting via DB Count

```python
async def check_rate_limit(db: AsyncSession, user_id: UUID) -> None:
    today = datetime.utcnow().replace(hour=0, minute=0, second=0, microsecond=0)
    count = (await db.execute(
        select(func.count(LlmUsage.id))
        .where(LlmUsage.user_id == user_id)
        .where(LlmUsage.created_at >= today)
    )).scalar_one()
    if count >= 10:
        raise HTTPException(429, "Daily limit of 10 LLM calls reached. Resets at midnight UTC.")
```

### AD-6: Error Handling Matrix

| Scenario | HTTP | Message |
|---|---|---|
| Anthropic API timeout / unavailable | 503 | "Assessment service temporarily unavailable. Please try again." |
| LLM returns unexpected response shape | 502 | "Failed to parse response. Please retry." |
| `chapter_id` not in Supabase | 404 | "Chapter '{id}' not found." |
| `student_answer` < 10 chars | 422 | Pydantic validation error |
| `chapter_ids` count < 2 or > 5 | 422 | Pydantic validation error |
| Rate limit exceeded | 429 | "Daily limit of 10 LLM calls reached. Resets at midnight UTC." |
| Non-pro tier | 403 | "This feature requires Pro plan. Upgrade at /pricing" |

## Database Migration Plan

**File**: `alembic/versions/<hash>_add_llm_usage_table.py`

```sql
CREATE TABLE llm_usage (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    feature     VARCHAR(20) NOT NULL CHECK (feature IN ('assessment','synthesis')),
    tokens_used INTEGER NOT NULL,
    cost_usd    FLOAT NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_llm_usage_user_date ON llm_usage(user_id, created_at);
```

**Cost formula** (`cost_tracker.py`):
```python
SONNET_INPUT_COST  = 3.00 / 1_000_000   # $3.00/M input tokens
SONNET_OUTPUT_COST = 15.00 / 1_000_000  # $15.00/M output tokens

def calculate_cost(usage) -> float:
    return round(
        usage.input_tokens * SONNET_INPUT_COST +
        usage.output_tokens * SONNET_OUTPUT_COST, 6
    )
```

## Security Plan

| Concern | Mitigation |
|---|---|
| Pro tier bypass | `require_pro` checks DB user tier on every request |
| API key exposure | `ANTHROPIC_API_KEY` only in `Settings`; never serialized to response |
| SSRF via chapter_id | `chapter_id` validated against `CHAPTER_METADATA` allowlist before fetch |
| Rate limit abuse | Count query before every LLM call; HTTP 429 on breach |
| JWT spoofing | Existing `get_current_user` dependency; `require_pro` builds on top |
